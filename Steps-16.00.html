The major issues in the data source were related to the way the data itself were handled or maintained. As a result, the following suggestions can be vital and be implemented to avoid the problems stated above in the future.
  

1.    Add source/Transformation Error Handler to redirect row data with “NULL” and/or Empty Cells back to Audit Folder and give access to the Recruiting and Staffing or the Clients’ Agents to update fields and extract the data on later package runtime.

2.    Add a failure constraint on the ETL Package that redirects the workflow to an alternate task and send an email to a Dev team to notify errors.

3.    Develop Data Mart for each Client that would help query execution performance and data security.
